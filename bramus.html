Hoe werkt het?

Zodra je een link klikt, word er een snapshot gemaakt van de huidige pagina. De volgende pagina wordt tegelijkertijd geladen onder de snapshot. Van de nieuwe pagina wordt ook een snapshot gemaakt en de twee snapshots worden onder elkaar gezet. Zodra dat allemaal klaar is word er een transitie gemaakt tussen de twee pagina‚Äôs.

Als je view transitions hebt, krijg je ‚Äúview-transition-name: root;‚Äù in je user-agent stylesheet.

view-transition-old en view-transition-new zijn dan de snapshots van de pagina.

Can I tweak the animations?

Ja dat kan. Standaard krijg je fade-out en fade-in. Dit zijn eigenlijk gewoon CSS animations, dus je kan ook de standaard animation properties aanpassen. Je kan de duration veranderen, maar je kan ook keyframes toevoegen.

Je moet wel onthouden dat de snapshot letterlijk een afbeelding is. Je kan dus niet de kleuren van elementen veranderen.

De nieuwe view is wel een live representatie van de pagina, maar het is wel flattened.

Je kan ook elementen echt animeren (zoals de positie van een element aanpassen) als je die elementen een view-transition-name geeft.

Normaal gesproken kan je niet meer klikken terwijl een view transition gaande is. Je kan hier omheen door pointer-events none te gebruiken, maar dan skipt de animatie wel. Dat is dus wel waar nog aan gewerkt kan worden.
