Leerdoelen

Na al deze projecten heb ik veel dingen geleerd, maar ook nog een aantal dingen die ik kan leren. Daarom stel ik voordat we de meesterproef ingaan drie leerdoelen voor mezelf:

Beter JavaScript leren kennen

Dit is de grootste voor mij en ééntje waar ik al sinds voordat ik met de minor ben begonnen mee zit. In het intake gesprek voor de minor had ik als algemene leerdoel om meer begrip te krijgen voor mijn code. Later heb ik ontdekt dat dit echt puur voor JavaScript geldt. Elke keer wanneer ik iets met JavaScript doe, probeer ik het zelf te maken, maar als het niet lukt ga ik al snel over naar chatgpt. Ik wil dus bij de meesterproef meer dingen doen met JavaScript om er meer intuïtief in te worden.

Toegankelijkheid als standaard behandelen

We hebben veel geleerd over toegankelijkheid in deze minor. Maar nogsteeds heb ik de neiging om toegankelijkheids dingen als laatst toe te voegen. Iets dat extra is. Ik wil mijn mindset daarin veranderen zodat het meer iets is wat ik standaard integreer in elk project waar ik aan werk. Dan voelt het sowieso minder als een irritatie dat ik later moet toevoegen, maar het kost ook minder tijd. Dat klinkt als een win win!

Meer vragen stellen

Dit is een soort van opvolger van "beter JavaScript leren". Dit probleem heb ik eigenlijk altijd al gehad. Ik wil altijd alles zelf oplossen, zelfs als ik het zelf niet altijd even goed kan. Hierdoor kom ik snel in de problemen met tijd. En hoe langer ik wacht met vragen, hoe minder ik durf te vragen, omdat ik het gevoel heb dat ik te ver achter loop of een domme vraag op het laatste moment stel. Ik moet me voor de meester proef minder zorgen maken dat ik docenten irriteer met mijn vragen, want daar zijn ze voor.
